# Prompt — Go-to-market: make Scripto viral on LinkedIn & X (phase-wise)

Paste into an AI chat (or hand to a marketing collaborator). Fill the `{brackets}`.

---

You are a senior developer-marketing strategist. Build me a **phase-wise launch plan + ready-to-post
content** to make **Scripto** go viral on **LinkedIn and X (Twitter)** and win as an open-source
project. Be specific, opinionated, and give me copy I can post, not vague advice.

## Product context
- **Scripto** — a free, open-source, 100% client-side **Markdown → pixel-perfect PDF** studio.
  Differentiators vs Notion/typical converters: true paginated PDF (running headers/footers, page
  numbers, repeating table headers, cover, TOC), **10+ document skins & themes**, **KaTeX math +
  Mermaid diagrams**, **offline PWA**, **privacy** (no server; optional zero-knowledge encryption),
  **full Arabic/RTL + i18n**, and **optional bring-your-own-key AI** (improve/summarize/translate).
- Audience: developers, technical writers, students/academics, job-seekers (résumé→PDF), indie hackers,
  Arabic-speaking users.
- Assets available: live demo `{url}`, GitHub repo `{url}`, logo, screenshots, and I can record short
  screen clips. Author handle: LinkedIn `{handle}`, X `{handle}`.

## The core viral loop to design around
Someone exports a beautiful PDF/diagram → others ask "what did you use?!" → one-click try (no signup)
→ they share their output too. Every asset should feed this loop.

## Deliverables (produce all)
1. **Positioning**: one crisp tagline + 3 alternates; a 1-sentence "what it is"; the 3 sharpest hooks
   ("Notion's export is mediocre — I built the opposite", "résumé→PDF that beats Word", "Arabic PDFs
   that actually look right"). A do/don't messaging guide.
2. **Phase plan (with timing):**
   - **Phase 0 — Prep (pre-launch):** demo polish, OG image, a 15–30s "type → beautiful PDF" screen
     recording, before/after (Notion vs Scripto) image, 3–5 skin screenshots, pinned README star CTA.
   - **Phase 1 — Soft launch / build-in-public:** teaser posts, "I'm building…", ask-for-feedback.
   - **Phase 2 — Big launch day:** coordinated LinkedIn post + X thread + Product Hunt + "Show HN" +
     r/opensource / r/selfhosted; sequencing and timing for each.
   - **Phase 3 — Sustain:** weekly cadence (feature spotlights, tips, user PDFs, milestones), collabs,
     newsletters, dev communities.
   - **Phase 4 — Amplify:** short-form video (X/LinkedIn/TikTok/YouTube Shorts), comparison content,
     use-case posts (résumé, README→PDF, academic, Arabic), contributor shoutouts.
3. **Ready-to-post copy:**
   - 5 **LinkedIn posts** (hook-first, story-driven, whitespace, 1 CTA, relevant hashtags) covering:
     launch, the "why I built it", a use-case (résumé), Arabic/RTL, and open-source/contributor call.
   - 3 **X threads** (strong first-tweet hook, 5–8 tweets, one idea per tweet, media cues, CTA + repo).
   - 10 standalone **X posts** / hooks for the content calendar.
   - A **Product Hunt** tagline + description + first comment; a **Show HN** title + body.
4. **A 2-week content calendar** (day-by-day: platform, format, hook, asset needed).
5. **Growth tactics**: how to seed the first 100 stars, who to tag/engage, comment strategy, reposting,
   cross-posting rules (never identical across platforms), timing windows, and 5 KPIs to track
   (stars, demo visits, signups-to-share, post impressions, referral traffic).
6. **Objection handling**: quick replies to "why not just Pandoc/Notion/Typora?", "is my data safe?",
   "does it work offline?".

## Style
- Practical and concrete. Give me copy I can paste. Match a credible builder voice: confident, specific,
  no hype-speak, no emoji spam (tasteful only). Optimize hooks for scroll-stopping first lines.
